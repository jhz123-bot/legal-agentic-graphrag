# 分层法律知识库清洗与标准化规则

本目录提供轻量数据清洗工具，目标是为分块、实体抽取、图构建与检索做准备。

## 清洗范围

- `laws` 层：`clean_laws.py`
- `cases` 层：`clean_cases.py`
- `faq` 层：`clean_faq.py`
- 通用标准化：`normalize_entities.py`

## 规则摘要

1. 文本清洗
- 删除空行
- 压缩重复标点（如“。。。” -> “。”）
- 删除常见 OCR 噪声字符（如“�”）

2. 法条名标准化
- `第二百六十四条`
- `刑法第二百六十四条`
- `《中华人民共和国刑法》第二百六十四条`

统一为：`刑法第二百六十四条`（在可识别法名时）

3. 实体别名标准化（示例）
- 房东 -> 出租人
- 租客 -> 承租人
- 盗窃行为 -> 盗窃罪

## 设计原则

- 轻量、可解释、可批处理
- 不改动核心业务逻辑，仅处理数据层
- 输出到 `data/legal_kb_cleaned/`，避免覆盖原始数据
